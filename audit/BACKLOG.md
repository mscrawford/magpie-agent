Added **Check 5 (DRIFT pre-flag)**: a single-line cite of an equation NAME (`qNN_*`, defined once per equations.gms) whose definition lands within +/-5 of the cited line but OUTSIDE the named equation's `[def..terminator]` span is pre-flagged as adjacent-block drift. The span computation (`eq_span_end`) is the load-bearing FP guard: docs legitimately cite a BODY line of an equation to pin a consumed/produced variable while naming that equation (verified corpus pattern: module_18.md `q50_nr_withdrawals`:40/41, module_30.md `q29_carbon`:31), so only cites landing OUTSIDE the equation are flagged. Same span guard retro-applied to the existing **CONTENT** check, eliminating a pre-existing FP (module_54.md `q11_cost_reg`:25 — defined at :15, but the cite correctly points at `vm_p_fert_costs` in the body). Positive controls: self-test assertions **D** (drift caught + clean control) and **E** (body-cite suppressed, both over/under-suppression directions); teeth-verified (neutering fails [D]). Corpus: **0 new FPs; advisory content-mismatch 2 -> 1** (then -> 0 once the comma-list work below fixed the lone remaining `v30_betr_missing` STRICT FP). The original R25 declarations.gms class was already covered by the earlier STRICT check.
- **Comma-list multi-cite parsing** (`scripts/check_gams_citations_impl.py`, `CITATION_RE`) — ✅ DONE 2026-06-05. `CITATION_RE` now captures comma-list continuations (`(?:, ?\d+(?:-\d+)?)+`), handling BOTH `:N,M` (~100) and `:N, M` (~64) spellings; an optional single space after the comma is allowed but a digit must follow (verified all 64 spaced matches are line-lists, not prose). Positive controls: self-test assertions **F** (comma-list STRICT sibling guard, the v30 FP class) and **G** (out-of-range continuation flagged, BOTH spellings); teeth-verified (neutering comma-list parsing fails [F]+[G]). Corpus: 2948 -> 2997 valid (de-merged distinct cites + newly-validated continuations), 0 errors, 0 content/strict advisories.
