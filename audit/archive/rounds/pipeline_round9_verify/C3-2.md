# Verification: C3-2 (DRIFT — README round count one behind validation_rounds.json)

**Verdict: UPHELD**

## Finding
README.md:7 says the flywheel "has matured to 47 rounds" but `audit/validation_rounds.json`
now holds 48 rounds. R48 was appended ~80 min after the README commit, same day, and never
propagated to the README narrative. Documented append-drift precedent class.

## STEP A — Mechanical citation check

```
$ wc -l README.md
     208 README.md                       # line 7 in range ✓

$ test -f audit/validation_rounds.json   # exists ✓

$ sed -n '7p' README.md
**Latest release:** v1.1.0 (2026-06-05) - see [CHANGELOG.md](CHANGELOG.md). The
semantic-validation flywheel has matured to 47 rounds (recent rounds 9.3-9.6 / 10,
periphery stabilized) and the docs are current with MAgPIE `develop`.

$ grep -n "matured to" README.md
7:**Latest release:** ... has matured to 47 rounds ...   # ✓ exact phrase present
```

Citation OK: file exists, line 7 in range, contains the claimed "matured to 47 rounds" token.

## STEP B — Reproduce

verify_cmd + independent second method:

```
$ python3 -c "import json;d=json.load(open('audit/validation_rounds.json'));r=d['rounds'];\
  print(len(r), max(x['round'] for x in r), d['cumulative_stats']['total_rounds'])"
48 48 48          # len=48, max round=48, cumulative_stats.total_rounds=48
```

README says 47, JSON says 48 on all three independent surfaces → they DISAGREE. Drift reproduces.

### Provenance of the drift (confirms append-drift class, not pre-existing off-by-one)

```
$ git log -1 --format="%H %ci" -- README.md
6d87471 ... 2026-06-05 12:51:36 +0200

$ git log -1 --format="%H %ci" -- audit/validation_rounds.json
1e0fb4f ... 2026-06-05 14:10:57 +0200      # 79 min AFTER README

$ git show 6d87471:audit/validation_rounds.json | python3 -c "import json,sys;\
  print(len(json.load(sys.stdin)['rounds']))"
47                                          # "47" was CORRECT at README commit time
```

So the README was accurate when written; R48 (a real `type:full` round, sonnet answerer /
opus auditor) was appended 79 min later and the README narrative was not updated. Exactly the
"drift every time a round is appended" failure mode the finding names.

**(a) No validator enforces the count** — confirmed:

```
$ grep -rn "matured to|47 rounds|README round count" scripts/     -> EXIT 1 (none)
$ grep -n "len(rounds)|count.*round" scripts/validate_consistency.sh scripts/selftest_validator.sh -> EXIT 1 (none)
```

Positive control (search machinery works in that dir):
```
$ grep -rln "validation_rounds" scripts/
scripts/selftest_validator.sh  scripts/probe_dedup_check.py  scripts/validate_consistency.sh
```
validate_consistency.sh:339-351 only asserts README *references* validation_rounds.json
(check_pass "points to ... live state") — it does NOT assert the round COUNT. Gap is real.

**(b) AGENT.md already defers totals (the precedent the fix cites)** — confirmed:
```
$ grep -n "cumulative_stats" AGENT.md
216: ... (see `audit/validation_rounds.json` cumulative_stats for current bug/round totals) ...

```
Deferral pattern exists at AGENT.md:216 and :418. The suggested "defer the count" fix mirrors
established repo practice and is sound. (Minor: fix text cites AGENT.md:216/418; verified — :216
is the verifiers/MANDATE line, :418 the QUALITY GUARD header; both defer round/bug totals.)

## Conclusion
Drift reproduces on all surfaces; provenance confirms it is the append-drift class (not an
off-by-one); the no-enforcing-validator and AGENT.md-deferral-precedent claims both verify.
Fix (prefer deferral) is correct. **UPHELD.** Not a removal — no live-dependent question.
