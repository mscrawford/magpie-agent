## Assessment of severity/fix
- **kind=untested is correct**: both are advisory detectors with brittle text extraction (1302-identifier unit table; arity-signature parsing) and zero positive control. A silent extraction regression (e.g., a declarations.gms format change, or a doc-markup change) would drop the canonical set or the arity signatures and the check would report "0 mismatch -> PASS" indistinguishably from genuinely-clean. This is exactly the failure class selftest_validator.sh was built to prevent (its own header documents the 2025-2026 silent-green incident).
- **MEDIUM severity is reasonable**: both are advisory (warning-only, never fail the suite), so a silent break degrades signal rather than blocking releases — lower stakes than an unguarded blocking check. But they are the two NEWEST cross-doc detectors and the only members of the cross-doc validator cohort (I1-I4 + R6 1a) without controls.
- **suggested_fix is sound and matches the established pattern**: add a `self_test()` that plants a known bug (units: doc claiming wrong unit for a known identifier; multi-section: fixture citing var(h) in one section, var(j) in another) and dispatch on `--self-test`, then append both basenames to `SELFTEST_SCRIPTS` in selftest_validator.sh. This is precisely how the 9 registered scripts already work (check_scaling.py:169-197 is the template).

No scope correction needed. UPHELD as written.
