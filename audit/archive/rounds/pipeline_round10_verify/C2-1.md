### FALSE claim 1 — "same property". They compute DIFFERENT properties.
Simulated both regexes on the actual cite lines:
```
Check 25 BARE_RE on 'modules/XX_.../equations.gms:123'      -> 0 matches
Check 25 BARE_RE on 'modules/80_optimization/*/input.gms:9' -> 0 matches
```
Check 25's `(?<![./])` lookbehind means slash-preceded `.gms` is NOT bare. These
are FULL-PATH cites; Check 25 never considered them bare. Reports 0 by CORRECT
classification, not by allowlist.

```
impl CITATION_RE on 'modules/XX_.../equations.gms:123'      -> gms_hint='/equations.gms'  (BARE-fallback)
impl CITATION_RE on 'modules/80_optimization/*/input.gms:9' -> gms_hint='/input.gms'       (BARE-fallback)
impl CITATION_RE on 'modules/42_water_demand/agr_sector_aug13/input.gms:9' -> gms_hint='modules/.../input.gms' (FILE-branch, resolves OK)
```
impl's prefix capture `modules/[\w/]+/` breaks on `...` (dots) and `*` (glob), so
placeholder/wildcard full-paths fall through to bare-fallback → `mod_num is None`
→ BARE. This is a REGEX-MISPARSE artifact, not bare-cite detection.

## Why CORRECTED, not UPHELD
The removal is safe, but the finding mis-frames WHAT is being removed:
- It is NOT a redundant copy of Check 25's property (different regex semantics:
  lexical-bare vs resolution-failure-after-misparse).
- The 22 are NOT allowlist-cleared bare cites; they're placeholder/wildcard
  full-path cites the impl regex can't parse.
- The "22 noise lines/run" gate impact is FALSE (gate suppresses them; 0 shown).
Acting on the finding-as-written ("merge away a duplicate of Check 25") would teach
the wrong model of the code. Correct scope below.

## Corrected finding
`check_gams_citations_impl.py:702-710` is an advisory-only branch that MIS-CLASSIFIES
placeholder/wildcard FULL-PATH .gms cites (`modules/XX_.../...`, `modules/NN_x/*/...`)
in non-module docs as "bare-basename" — because CITATION_RE's `modules/[\w/]+/`
prefix capture breaks on `.`/`*`. It is NOT a duplicate of Check 25 (which correctly
treats these as non-bare). It emits 22 advisories ONLY on direct invocation; the
validate_consistency.sh Check-17 harness already suppresses them (0 in the gate).
Deleting 702-710 (or, better, narrowing CITATION_RE / FILE-branch to recognize
placeholder paths) is SAFE — advisory-only, no external consumer, self-test green —
and removes a latent false-positive source. But there is NO gate-visible noise to
eliminate, and the "redundant with Check 25" rationale is incorrect.
