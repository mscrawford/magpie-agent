## The finding that reorganizes the whole picture

The downloaded `modules/32_forestry/input/npi_ndc_aff_pol.cs3` is **an all-zero dummy file**, by design.

```
$ head -2 modules/32_forestry/input/npi_ndc_aff_pol.cs3
* description: Dummy file for NPI/INDC policies
* unit: dummy (none)

$ awk -F',' 'NR>5 { if ($3+0!=0) a++; if ($4+0!=0) b++; if ($5+0!=0) c++; n++ } \
    END { printf "data rows=%d none-nonzero=%d npi-nonzero=%d ndc-nonzero=%d\n", n,a+0,b+0,c+0 }' \
    modules/32_forestry/input/npi_ndc_aff_pol.cs3
data rows=6400  none-nonzero=0  npi-nonzero=0  ndc-nonzero=0
```

Every policy column — including `npi`, the **default** (`config/default.cfg:1026`, `input.gms:9`) — is zero in the shipped input. The maintainers say so themselves:

```
config/default.cfg:119: # * (ifneeded): NPI/NDC recalculation will only be executed if current input files are zero
config/default.cfg:120: # *             and policy switches (c32_aff_pol, c35_ad_pol) are set to "npi" or "ndc".
```

This is why the recalc guard's **primary** trigger is `all(aff_pol == 0) && (c32_aff_policy != "none")` (`scripts/start_functions.R:385`) — it fires on **every** fresh-download run for **every** non-`none` policy. The `affexp_missing`/`ndcdelay_missing` tests (`:380-383`) are *secondary* triggers that additionally catch a **previously-regenerated, non-zero** file that predates the new columns.

The doc names only the secondary triggers, and therefore builds a story in which a 3-column file is an anomaly ("a stale artifact from a previous run") rather than what it actually is: **the pristine, expected, freshly-downloaded input**.

## Meta

The doc's own §"🚩 How this note got written" lesson — *"a file under `modules/*/input/` is **not** necessarily an input you were given"* — is right, and the audit confirms every one of its factual premises. The over-correction is that the doc then swung to the opposite absolute: *"not an input you download."* The truth is the boring conjunction — **it is a downloaded input AND a run-time product**, and the reason the download looks empty is that it ships as a **dummy**. The transferable lesson is one notch stronger than the doc's: *find the producer* did not mean *the R layer is the producer instead of the download* — it meant **enumerate all producers, then check which one wrote the bytes in front of you** (here: the `* origin:` header is the discriminator, and it was sitting in line 3 of the file the whole time).
