## 1. Parameters that determine tree-cover carbon density on cropland

**Parameter**: `p29_carbon_density_ac(t,j,ac,ag_pools)` — set in `modules/29_cropland/detail_apr24/preloop.gms:45-49`, per `modules/module_29.md` (Algorithm 6, lines 762-818).

**Source, switch-dependent** (`s29_treecover_plantation`, default = 0, `input.gms:20`):

- **`s29_treecover_plantation = 0` (default)**:
  ```gams
  p29_carbon_density_ac(t,j,ac,ag_pools) = pm_carbon_density_secdforest_ac_uncalib(t,j,ac,ag_pools);
  ```
- **`s29_treecover_plantation = 1`**:
  ```gams
  p29_carbon_density_ac(t,j,ac,ag_pools) = pm_carbon_density_plantation_ac_uncalib(t,j,ac,ag_pools);
  ```

Both source parameters — `pm_carbon_density_secdforest_ac_uncalib` and `pm_carbon_density_plantation_ac_uncalib` — are **produced in Module 52 (Carbon)**, `modules/52_carbon/normal_dec17/start.gms:43-44`, and copied into Module 29 wholesale (M29 does not compute either curve itself). 🟢 (per `modules/module_52.md` lines 815-816, 454-478, itself citing `start.gms:43-44`)

**Why the *uncalibrated* curves specifically**: as of MAgPIE PR #869 (2026-04-20, commit `75d7ee167`), Module 52 gained a growing-stock calibration system (`s52_growingstock_calib`, default = 1) that overwrites the *calibrated* `pm_carbon_density_secdforest_ac` / `pm_carbon_density_plantation_ac` (vegc pool) to match FAO FRA 2025 growing-stock targets for **existing** forest. Tree cover on cropland represents **new establishment** (analogous to afforestation), so Module 29 deliberately reads the pre-calibration snapshot instead — documented explicitly in `module_29_notes.md` line 13 and `module_52.md` §2.C. 🟡

Both curves are ultimately Chapman-Richards growth functions (macro `m_growth_vegc`, `core/macros.gms:18`, per `module_52.md` line 82-85):
```
vegc(ac) = S + (A - S) * (1 - exp(-k * ac*5))**m
```
with `S = 0`, `A = fm_carbon_density(t,j,"secdforest","vegc")` (the LPJmL secondary-forest asymptote — **the same asymptote regardless of the plantation/secdforest switch**), and climate-class-weighted `k`, `m` from `f52_growth_par.csv` (natveg or plantation parameter set depending on the switch). Litter carbon (`litc`) uses a separate linear-to-20-year macro (`m_growth_litc_soilc`, `core/macros.gms:20`). Per `module_29.md`'s R58 correction (lines 784-816): because both curves share the same asymptote `A`, the plantation curve is **faster, never lower** at any age class — there is no "plantation has lower final carbon density" trade-off in the model.

**Growth-curve selection switch**: `s29_treecover_plantation` (0 = natveg/secdforest curve [default], 1 = plantation curve) — `input.gms:20`. 🟡

## 2. Age-class dimension used

The dimension is **`ac`** (age class), the standard 5-year-increment MAgPIE age-class set. `p29_carbon_density_ac` is indexed `(t,j,ac,ag_pools)`, and the state variable `v29_treecover(j,ac)` carries the same `ac` dimension.

Within `ac`, two subsets matter for the carbon equation:
- **`ac_est`** — establishment age classes (newly planted this timestep; free decision variable, `v29_treecover.lo/.up` in `presolve.gms:78-79`)
- **`ac_sub`** — all other (non-establishment) age classes, **fixed** to the previous timestep's age-shifted state (`v29_treecover.fx(j,ac_sub) = pc29_treecover(j,ac_sub)`, `presolve.gms:78-82`) — the optimizer can plant new trees but cannot remove existing tree cover.

`ag_pools` is the above-ground-pools subset of `c_pools` (vegc, litc only — no age-class soilc dimension for tree cover). 🟢 (per `modules/module_29.md` lines 140-159, 263)

## 3. Equation that turns density into stock: `q29_carbon`

**Location**: `modules/29_cropland/detail_apr24/equations.gms:38-42` (per `modules/module_29.md` lines 227-236).

```gams
q29_carbon(j2,ag_pools,stockType)..
  vm_carbon_stock(j2,"crop",ag_pools,stockType) =e=
    vm_carbon_stock_croparea(j2,ag_pools)
    + vm_fallow(j2) * sum(ct, fm_carbon_density(ct,j2,"crop",ag_pools))
    + m_carbon_stock_ac(v29_treecover, p29_carbon_density_ac, "ac", "ac_sub");
```

The tree-cover contribution is entirely inside the third term, via the macro `m_carbon_stock_ac` (`core/macros.gms:104-106`, quoted verbatim in `modules/module_29.md` lines 253-256):

```gams
$macro m_carbon_stock_ac(land,carbon_density,sets,sets_sub) \
   sum((&&sets),     land(j2,&&sets)     * sum(ct, carbon_density(ct,j2,&&sets,ag_pools)))$(sameas(stockType,"actual")) + \
   sum((&&sets_sub), land(j2,&&sets_sub) * sum(ct, carbon_density(ct,j2,&&sets_sub,ag_pools)))$(sameas(stockType,"actualNoAcEst"))
```

With the macro arguments substituted (`land = v29_treecover`, `carbon_density = p29_carbon_density_ac`, `sets = ac`, `sets_sub = ac_sub`), the two branches are:
- `stockType = "actual"`: `sum(ac, v29_treecover(j2,ac) * p29_carbon_density_ac(ct,j2,ac,ag_pools))` — **all** age classes, including newly established (`ac_est`)
- `stockType = "actualNoAcEst"`: `sum(ac_sub, v29_treecover(j2,ac_sub) * p29_carbon_density_ac(ct,j2,ac_sub,ag_pools))` — **excludes** establishment-year age classes

This `$(sameas(...))` construct is a GAMS branch-selection idiom, not an accumulation — only one term is active per `stockType` member, so there is no double-counting. `stockType` has exactly two members, `actual` and `actualNoAcEst` (declared `modules/56_ghg_policy/price_aug22/sets.gms:212-213`) — **not** "actual vs reference"; a corrected mislabel flagged in `module_29.md` lines 267-272. 🟢 (per `modules/module_29.md`, which states this was cross-verified against `core/macros.gms:104-106` in R58)

So the full cropland carbon stock = croparea carbon (from M30, via `vm_carbon_stock_croparea`) + fallow carbon (fallow area × cropland `fm_carbon_density`) + tree-cover carbon (age-class area × age-class-specific density via the macro above).

## 4. Declaration vs. population of `vm_carbon_stock`

**Declares**: `vm_carbon_stock(j,land,c_pools,stockType)` is declared in **Module 56 (GHG Policy)**, realization `price_aug22`, at `modules/56_ghg_policy/price_aug22/declarations.gms:34`. 🟡 (per `modules/module_56.md` line 585, cross-confirmed in `modules/module_52.md` line 415-424, which independently states "Module 52 uses interface variables declared in Module 56")

**Populates** (per-land-type slices; `modules/module_56.md` line 584 and `modules/module_52.md` line 424, both consistent):
- **Module 29 (Cropland)** — `"crop"` slice, via `q29_carbon` above (folds in Module 30's `vm_carbon_stock_croparea`, which M30 computes separately and does not write to `vm_carbon_stock` itself)
- **Module 31 (Pasture)** — `"past"` slice
- **Module 32 (Forestry)** — `"forestry"` slice (plantations)
- **Module 34 (Urban)** — `"urban"` slice, fixed to 0: `vm_carbon_stock.fx(j,"urban",ag_pools,stockType) = 0` (`modules/34_urban/exo_nov21/presolve.gms:8`)
- **Module 35 (Natural Vegetation)** — `"primforest"`, `"secdforest"`, `"other"` slices
- **Module 59 (SOM)** — the **`soilc`** pool, for **all** land types (cutting across the land-type slices above), via `q59_carbon_soil` (`modules/59_som/cellpool_jan23/equations.gms:61-64`)
- **Module 58 (peatland)** — does **NOT** populate `vm_carbon_stock`; peatland carbon flows through a separate emissions path (`q58_peatland_emis` → `vm_emissions_reg(i,"peatland",poll58)`), not the stock-change mechanism described here.

So for the **cropland slice specifically**, the sole populator is **Module 29** (`q29_carbon`), which itself draws on Module 30 (croparea carbon) and Module 59 (soilc, folded in orthogonally by pool rather than by this equation). 🟢 (`modules/module_29.md`, `modules/module_52.md`, `modules/module_56.md` — mutually consistent)

## 5. How the cropland carbon stock reaches Module 52's CO2 accounting

Module 52's sole equation, `q52_emis_co2_actual` (`modules/52_carbon/normal_dec17/equations.gms:16-19`, per `modules/module_52.md` lines 298-312, 776-826):

```gams
q52_emis_co2_actual(i2,emis_oneoff)..
  vm_emissions_reg(i2,emis_oneoff,"co2_c") =e=
    sum((cell(i2,j2),emis_land(emis_oneoff,land,c_pools)),
      (pcm_carbon_stock(j2,land,c_pools,"actual") - vm_carbon_stock(j2,land,c_pools,"actual"))
      / m_timestep_length
    );
```

For the cropland/tree-cover contribution, `emis_oneoff` includes `crop_vegc` and `crop_litc` (mapped via `emis_land` to `land="crop"`, `c_pools={vegc,litc}`, `core/sets.gms:314-318,332-335`). Module 52 reads `vm_carbon_stock(j2,"crop",ag_pools,"actual")` — i.e. the **`"actual"`** `stockType` slice, meaning tree cover's establishment-year age classes (`ac_est`) **are** included in what M52 uses for CO2 accounting. It differences this against `pcm_carbon_stock`, the same slice carried forward from the previous timestep (carry-forward for the above-ground `ag_pools` happens in Module 56 postsolve, `modules/56_ghg_policy/price_aug22/postsolve.gms:8`; the `soilc` pool is carried forward separately by Module 59, `modules/59_som/cellpool_jan23/postsolve.gms:13`), and divides by `m_timestep_length` to get an annualized flow into `vm_emissions_reg(i2,"crop_vegc"/"crop_litc","co2_c")`. 🟢 (per `modules/module_52.md`, cross-verified against `equations.gms:16-19` per that doc's own citation)

**Important asymmetry, documented in `cross_module/carbon_balance_conservation.md` line 101**: Module 52's *reported* CO2 accounting reads the `"actual"` slice (all age classes, including `ac_est`), but Module 56's *carbon-pricing* equation `q56_emis_pricing_co2` (`modules/56_ghg_policy/price_aug22/equations.gms:19-22`) reads `vm_carbon_stock(...,%c56_carbon_stock_pricing%)`, which **defaults to `actualNoAcEst`** (`config/default.cfg:1835`) — excluding establishment-year tree-cover area from what actually gets priced. M56 does **not** consume M52's `vm_emissions_reg(...,"co2_c")` output at all for pricing; M52 and M56 are **parallel, independent readers** of `vm_carbon_stock`/`pcm_carbon_stock`, not a serial producer→consumer chain. M56 reads M52's `vm_emissions_reg(...,"co2_c")` only for **reporting** (`modules/56_ghg_policy/price_aug22/postsolve.gms:27`). 🟡 (per `modules/module_52.md` lines 411-443, 741-743, and `cross_module/carbon_balance_conservation.md` lines 99-101, 583)

Also worth flagging for the tree-cover case specifically: at stock defaults (`c56_emis_policy = "reddnatveg_nosoil"`, `config/default.cfg:1828`), `crop_vegc`, `crop_litc`, and `crop_soilc` are **zeroed out** in `modules/56_ghg_policy/price_aug22/input/f56_emis_policy.csv:114` — meaning even though the CO2 physically flows through `vm_carbon_stock` and into `q52_emis_co2_actual`'s reported emissions, it earns **no carbon-price revenue** in Module 56's objective-function pricing under default settings. This is stated explicitly in `modules/module_29.md`'s capability-vs-default box (lines 52-56) as one of the reasons planting tree cover is economically dominated at defaults. 🟡

## Summary chain (cropland tree-cover carbon → CO2 accounting)

1. `s29_treecover_plantation` selects a Chapman-Richards curve (`m_growth_vegc`, `core/macros.gms:18`) computed in **Module 52** (`start.gms:17` or `:28`) and preserved uncalibrated (`start.gms:43-44`) → copied into **Module 29**'s `p29_carbon_density_ac(t,j,ac,ag_pools)` (`29_cropland/detail_apr24/preloop.gms:45-49`).
2. **Module 29**'s `v29_treecover(j,ac)` (area by age class) × `p29_carbon_density_ac` is summed via macro `m_carbon_stock_ac` (`core/macros.gms:104-106`) inside equation `q29_carbon` (`29_cropland/detail_apr24/equations.gms:38-42`), writing the `"crop"` slice of `vm_carbon_stock(j,land,c_pools,stockType)` — **declared in Module 56** (`56_ghg_policy/price_aug22/declarations.gms:34`).
3. **Module 52**'s `q52_emis_co2_actual` (`52_carbon/normal_dec17/equations.gms:16-19`) differences `vm_carbon_stock(...,"actual")` against the prior-timestep `pcm_carbon_stock(...,"actual")` for `land="crop"`, pools `vegc`/`litc` (source `crop_vegc`/`crop_litc` in `emis_oneoff`), divides by `m_timestep_length`, and writes `vm_emissions_reg(i,"crop_vegc"/"crop_litc","co2_c")`.
4. In parallel (not downstream), **Module 56** independently recomputes CO2 pricing from the same stocks via `q56_emis_pricing_co2`, but from the `actualNoAcEst` slice by default — excluding establishment-year tree-cover area from the priced quantity, and zeroing crop pools entirely under the default `reddnatveg_nosoil` emission policy.
