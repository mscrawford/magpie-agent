## Equations defined in equations.gms

Module 14 defines exactly **2 equations** (per `module_14.md` §2, "Module 14 implements only 2 equations, but relies on extensive preloop calibration to prepare the input data"):

### 1. `q14_yield_crop` — Crop Yield Calculation

**File:** `equations.gms:14-16` (doc also cites the fuller block as `equations.gms:14-20`)

```gams
q14_yield_crop(j2,kcr,w) ..
 vm_yld(j2,kcr,w) =e= sum(ct,i14_yields_calib(ct,j2,kcr,w)) *
                         vm_tau(j2,"crop") / sum((cell(i2,j2), supreg(h2,i2)), fm_tau1995(h2));
```

Calculates cellular crop yields (`vm_yld(j2,kcr,w)`) as the calibrated baseline yield (`i14_yields_calib`, produced in the preloop calibration stages) scaled by the ratio of the current technological-change factor (`vm_tau(j2,"crop")`, from Module 13) to the 1995 baseline τ (`fm_tau1995(h2)`, looked up via the `cell(i2,j2)` → `supreg(h2,i2)` spatial mapping to the super-region level). Since the `f_btc2` change, `vm_tau` is defined at cluster level (j) rather than super-region level (h).

### 2. `q14_yield_past` — Pasture Yield Calculation

**File:** `equations.gms:35-39` (doc also cites the fuller block as `equations.gms:24-39`)

```gams
q14_yield_past(j2,w) ..
 vm_yld(j2,"pasture",w) =e=
 sum(ct,(i14_yields_calib(ct,j2,"pasture",w))
 * sum(cell(i2,j2),pm_past_mngmnt_factor(ct,i2)))
 * (1 + s14_yld_past_switch*(sum((cell(i2,j2), supreg(h2,i2)), pcm_tau(j2, "crop")/fm_tau1995(h2)) - 1));
```

Calculates pasture yields (`vm_yld(j2,"pasture",w)`) from the calibrated pasture baseline, multiplied by an exogenous pasture-management factor (`pm_past_mngmnt_factor`, driven by Module 70/livestock demand), plus a spillover term: a fraction (`s14_yld_past_switch`, default 0.25 — `input.gms:20`) of the *previous-timestep* crop-sector τ growth (`pcm_tau(j2,"crop")` relative to `fm_tau1995`) partially carries over into pasture productivity. Note pasture uses the **previous** timestep's crop τ (`pcm_tau`), not the current one (`vm_tau`), reflecting a lag in knowledge transfer from crop to pasture management.

Both equations output to the same interface variable `vm_yld(j,kve,w)` (declared `declarations.gms:27`), consumed downstream by Module 30 (Croparea) and Module 31 (Pasture) for production = area × yield.

No other `q14_*` equations are documented — all remaining Module-14 complexity (the 6-stage yield calibration system, growing-stock conversion for timber) lives in `preloop.gms` and `presolve.gms` as parameter assignments, not as declared GAMS equations.
