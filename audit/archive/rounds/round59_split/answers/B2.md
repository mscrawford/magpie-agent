## 1. How Module 35 computes carbon stocks per land pool

Module 35 tracks three land pools — primary forest, secondary forest, and other land — and computes `vm_carbon_stock(j,land,ag_pools,stockType)` for each via three separate equations in `equations.gms` (🟡 module_35.md, cites `equations.gms:42-55`):

**Primary forest** — `q35_carbon_primforest` (`equations.gms:42-44`):
```gams
vm_carbon_stock(j2,"primforest",ag_pools,stockType) =e=
  m_carbon_stock(vm_land,fm_carbon_density,"primforest");
```
Uses the non-age-class macro `m_carbon_stock` (declared `core/macros.gms:99-101`, per `audit/archive/rounds/round23_answers.md:277`) because primary forest carries no age-class dimension — it is assumed uniformly mature. Density source: `fm_carbon_density(t,j,"primforest",ag_pools)`, the global LPJmL-derived carbon density parameter (Module 52).

**Secondary forest** — `q35_carbon_secdforest` (`equations.gms:49-51`):
```gams
vm_carbon_stock(j2,"secdforest",ag_pools,stockType) =e=
  m_carbon_stock_ac(v35_secdforest,p35_carbon_density_secdforest,"ac","ac_sub");
```
Uses the age-class macro `m_carbon_stock_ac` (declared `core/macros.gms:104-106`), which — per its verified body (`audit/archive/rounds/round58_hubs/module_29_refute.md:25-27`, quoting `core/macros.gms:104-106`):
```gams
$macro m_carbon_stock_ac(land,carbon_density,sets,sets_sub) \
   sum((&&sets), ...)$(sameas(stockType,"actual")) + \
   sum((&&sets_sub), ...)$(sameas(stockType,"actualNoAcEst"))
```
is a **mutually-exclusive dispatch on `stockType`**, not an additive double-sum: when `stockType = "actual"` it sums area × density over the full `ac` set; when `stockType = "actualNoAcEst"` it sums only over `ac_sub` (excluding establishment classes). Area comes from `v35_secdforest(j,ac)`; density comes from the module-local blended parameter `p35_carbon_density_secdforest(t,j,ac,ag_pools)` (see §2 below for the blend).

**Other land** — `q35_carbon_other` (`equations.gms:53-55`):
```gams
vm_carbon_stock(j2,"other",ag_pools,stockType) =e=
  m_carbon_stock_ac(vm_land_other,p35_carbon_density_other,"othertype35,ac","othertype35,ac_sub");
```
Same age-class macro, but iterated over both `othertype35` (`"othernat"`, `"youngsecdf"`) and `ac`, using `p35_carbon_density_other(t,j,ac,ag_pools)` — set from `pm_carbon_density_other_ac` for `othernat`, and (as of 2026-04-20) from the uncalibrated secdforest curve `pm_carbon_density_secdforest_ac_uncalib` for `youngsecdf` (`module_35.md:356`).

All three equations are dimensioned over `ag_pools` — i.e. `vegc` and `litc` only. **Soil carbon (`soilc`) is not computed by Module 35 at all**: it is a separate slice of `vm_carbon_stock` owned by Module 59 (SOM) for every land type (`module_52.md:424,718`, `agent/helpers/verifiers.md:302`).

## 2. Carbon-density parameters and their dimensions

All age-class carbon densities are computed in Module 52 (`start.gms`), not Module 35 — Module 35 only consumes them (plus one Module-35-local blended parameter). 🟡 module_52.md, module_35.md.

| Parameter | Dimensions | Producer | Consumed by M35 for |
|---|---|---|---|
| `fm_carbon_density(t_all,j,land,c_pools)` | t_all × j × land(7 types incl. primforest) × c_pools(vegc,litc,soilc) | Module 52, from LPJmL (`lpj_carbon_stocks.cs3`) | Primary forest (`q35_carbon_primforest`) |
| `pm_carbon_density_secdforest_ac(t_all,j,ac,ag_pools)` | t_all × j × ac(62) × ag_pools(vegc,litc) | Module 52 `start.gms:28,31`, vegc overwritten in `preloop.gms` when `s52_growingstock_calib=1` (default ON) — FRA-2025-calibrated Chapman-Richards curve | The "existing/managed" leg of the blended `p35_carbon_density_secdforest` (`presolve.gms:248-252`) |
| `pm_carbon_density_secdforest_ac_uncalib(t_all,j,ac,ag_pools)` | same | Module 52 `start.gms:43`, saved before the preloop calibration overwrite — uncalibrated Braakhekke et al. natveg curve | The 20 tC/ha maturation test (`presolve.gms:117`), youngsecdf carbon density (`presolve.gms:242`), and the "natural-origin" leg of the blend (`presolve.gms:249-252`) |
| `pm_carbon_density_other_ac(t_all,j,ac,ag_pools)` | same | Module 52 `start.gms:48,51` — NOT calibrated (no FRA target for "other" land) | `othernat` carbon density directly |
| `p35_carbon_density_secdforest(t,j,ac,ag_pools)` | t × j × ac × ag_pools | **Module 35-local** (`presolve.gms:248-252`), area-weighted blend of the two secdforest curves above | Feeds `q35_carbon_secdforest` directly |
| `p35_carbon_density_other(t,j,othertype35,ac,ag_pools)` | t × j × othertype35 × ac × ag_pools | Module 35-local; `othernat` slice from `pm_carbon_density_other_ac`, `youngsecdf` slice from `pm_carbon_density_secdforest_ac_uncalib` (`presolve.gms:241-242`) | Feeds `q35_carbon_other` |

**Blend formula** for `p35_carbon_density_secdforest` (`presolve.gms:248-252`, 🟡 module_35.md:344-354, "NEW 2026-04-20" natural-origin tracking):
```gams
p35_carbon_density_secdforest(t,j,ac,ag_pools) = pm_carbon_density_secdforest_ac(t,j,ac,ag_pools);
p35_carbon_density_secdforest(t,j,ac,ag_pools)$(pc35_secdforest(j,ac) > 1e-10) =
  pm_carbon_density_secdforest_ac(t,j,ac,ag_pools)
  - (pm_carbon_density_secdforest_ac(t,j,ac,ag_pools) - pm_carbon_density_secdforest_ac_uncalib(t,j,ac,ag_pools))
    * pc35_secdforest_natural(j,ac) / pc35_secdforest(j,ac);
```
i.e., density is the FRA-calibrated curve minus a correction proportional to the natural-origin area share `pc35_secdforest_natural(j,ac)/pc35_secdforest(j,ac)`. Where that share is 0, density = FRA-calibrated; where it is 1, density = uncalibrated.

**Underlying growth functions** (Module 52, not Module 35):
- vegc: Chapman-Richards, macro `m_growth_vegc(S,A,k,m,ac) = S + (A-S)*(1-exp(-k*(ac*5)))**m` (`core/macros.gms:18`, quoted `module_52.md:82-92`), with `S=0`, `A=fm_carbon_density(...,"secdforest","vegc")` (or `"other"`), and `k,m` climate-class-weighted Chapman-Richards parameters. Since 2026-04-20 (default), the secdforest `k` is regionally calibrated via 25-iteration bisection to FAO FRA 2025 growing-stock targets (`module_52.md:218-296`) — this is what produces the calibrated/uncalibrated split.
- litc: linear-to-equilibrium macro `m_growth_litc_soilc` (`core/macros.gms:20`, despite the name applied only to litc in Module 52) over a 20-year (4-age-class) horizon.
- soilc: no age-class function anywhere — read directly from `fm_carbon_density(...,soilc)` / handled entirely by Module 59.

## 3. How Module 28 age classes enter secondary-forest carbon accounting

Module 28 is a **pure data provider** (0 equations, 0 variables) that defines the `ac` set structure and two dynamic sub-sets (`module_28.md:12-16`). Its role in M35 carbon accounting is threefold:

1. **Initial age-class distribution** (one-time, preloop): `p35_secdf_ageclass(j,ac) = im_forest_ageclass(j,ac)` (`modules/35_natveg/pot_forest_may24/preloop.gms:20`, per `module_28.md:222-224`) — initializes secondary-forest area per age class from the GFAD (Global Forest Age Dataset, Poulter et al. 2019) satellite distribution at model start (year ~1995), gated by the default switch `s35_secdf_distribution = 2` (`module_35.md:850-853`).
2. **`ac_est` / `ac_sub` dynamic sets**, recomputed every timestep from `m_yeardiff_forestry(t)` (`modules/28_ageclass/oct24/presolve.gms:9-13`, quoted `module_28.md:145-171`):
   ```gams
   ac_est(ac) = yes$(ord(ac) <= (m_yeardiff_forestry(t)/5))
   ac_sub(ac) = yes$(ord(ac) > (m_yeardiff_forestry(t)/5))
   ```
   Module 35 uses `ac_sub` in the carbon macro's `"actualNoAcEst"` dispatch branch (excludes establishment-age area from the stock, `equations.gms:51` sets_sub argument = `"ac_sub"`) and uses `ac_est` for redistributing new secondary forest / disturbance-recovered area evenly across establishment classes (`equations.gms:228-229`, `presolve.gms:36-39`).
3. Age-class *shifting* each timestep is Module-35-local logic (`presolve.gms:84-97`, not Module 28): `s35_shift = m_timestep_length_forestry/5`; `p35_secdforest(t,j,ac)$(ord(ac)>s35_shift) = pc35_secdforest(j,ac-s35_shift)`, with overflow accumulating in `acx`. Module 28 supplies the set structure (`ac`, 62 members) that this shift operates over, but does not perform the shift itself.

Module 28 does **not** provide carbon densities — those are Module 52's job (§2). Module 28's only carbon-relevant output besides the `ac` structure is `im_forest_ageclass(j,ac)`, which Module 52 also reads directly (not via Module 35) to weight the secdforest growing-stock calibration bisection (`modules/52_carbon/normal_dec17/preloop.gms:53-59`, per `module_28.md:228,442`).

## 4. Exact `ac` set members and count

**62 members** (`module_28.md:22-28`, "⚠️ Age-Class Range Summary" box; corroborated `module_35.md:134`):

```
ac0, ac5, ac10, ac15, ac20, ..., ac295, ac300, acx
```
- `ac0` through `ac300`: 61 regular classes in 5-year increments spanning 0–300 years (i.e. `ac0, ac5, ac10, ..., ac300` = 61 elements, since (300-0)/5 + 1 = 61).
- `acx`: one catch-all absorbing class for mature/primary-adjacent forest ≥300 years (per module_35.md) / ≥150 years in the raw GFAD class15 mapping (per module_28.md:26,87,119 — GFAD's own class15 boundary is 150+ years, but MAgPIE's `ac` set numeric range independently runs to ac300 before folding into `acx`).
- Total: 61 + 1 = **62 elements**, matching the explicit "Total: 62 elements" statement in `module_28.md:27` and "62 age classes" in `module_35.md:134`.

The doc flags this explicitly as a truncation trap: "not fewer — do NOT truncate this range" (`module_28.md:27`).

Which subsets Module 35 actually iterates over for secondary-forest carbon:
- `q35_carbon_secdforest` sums over the full `ac` set (62 members) when `stockType="actual"`, or over `ac_sub` (timestep-dependent subset, e.g. `{ac10,...,acx}` for a 10-year step with `ac_est={ac0,ac5}`) when `stockType="actualNoAcEst"`.
- The 20 tC/ha maturation test (`presolve.gms:116`) explicitly excludes `acx`: `p35_maturesecdf(t,j,ac)$(not sameas(ac,"acx"))`.

## 5. How Module 35's carbon contribution reaches Module 52

`vm_carbon_stock(j,land,ag_pools,stockType)` is **declared in Module 56** (GHG Policy), not Module 52 or Module 35 (`module_52.md:415-424`, "Provider: Modules 29, 31, 32, 34, 35, and 59 populate `vm_carbon_stock` by land type"). Module 35 is one of several land modules that write into disjoint `land`-slices of this shared interface variable — it writes the `"primforest"`, `"secdforest"`, and `"other"` slices via its three carbon equations (§1).

Module 52 does **not** receive a variable "from" Module 35 through a direct M35→M52 interface parameter; instead **both modules write into and read from the shared `vm_carbon_stock` / `pcm_carbon_stock` interface**:
- Module 35's `q35_carbon_primforest`, `q35_carbon_secdforest`, `q35_carbon_other` populate the primforest/secdforest/other slices of `vm_carbon_stock`.
- Module 52's sole equation, `q52_emis_co2_actual` (`modules/52_carbon/normal_dec17/equations.gms:16-19`, quoted `module_52.md:820-824`):
  ```gams
  vm_emissions_reg(i2,emis_oneoff,"co2_c") =e=
    sum((cell(i2,j2),emis_land(emis_oneoff,land,c_pools)),
      (pcm_carbon_stock(j2,land,c_pools,"actual") - vm_carbon_stock(j2,land,c_pools,"actual"))
      / m_timestep_length
    );
  ```
  sums over `emis_land(emis_oneoff,land,c_pools)` — which includes the mappings `primforest_vegc→(primforest,vegc)`, `secdforest_vegc→(secdforest,vegc)`, `other_vegc→(other,vegc)`, and their litc/soilc counterparts (`module_52.md:330-341`) — reading exactly the slices Module 35 (and Module 59 for soilc) wrote. The previous-timestep value `pcm_carbon_stock` for the above-ground pools is carried forward each timestep in **Module 56's** postsolve (`modules/56_ghg_policy/price_aug22/postsolve.gms:8`), not Module 52's (`module_52.md:431`).

So the M35→M52 "provides to" relationship documented at `module_35.md:882-883` ("To Module 52 (Carbon): `vm_carbon_stock(j,land_natveg,ag_pools,stockType)` — Carbon stocks") is mediated entirely through this shared variable declared by Module 56, with Module 52 differencing consecutive timesteps to derive `vm_emissions_reg`, which Module 56 in turn separately re-derives for carbon pricing (`q56_emis_pricing_co2`, which recomputes `pcm_carbon_stock - vm_carbon_stock` independently rather than reading Module 52's output — `module_52.md:443,767`).

## 6. Contrast with Module 32 (forestry) age-class carbon

Module 32 manages plantation/afforestation carbon with the **same macro** but different set structure and, critically, **type-dependent density sourcing**:

**`q32_carbon`** (`modules/32_forestry/dynamic_may24/equations.gms:108-109`, per `module_32.md:373-376`):
```gams
q32_carbon(j2,ag_pools,stockType) .. vm_carbon_stock(j2,"forestry",ag_pools,stockType) =e=
  m_carbon_stock_ac(v32_land,p32_carbon_density_ac,"type32,ac","type32,ac_sub");
```
This is structurally identical to `q35_carbon_secdforest` (same `m_carbon_stock_ac` macro, same actual/actualNoAcEst dispatch) but sums over an additional dimension: `type32` — MAgPIE's three plantation types (`sets.gms:16-17`, `module_32.md:41-136`):
- `plant` (commercial timber plantations): density from `pm_carbon_density_plantation_ac` — **calibrated** fast-growth curve (`module_32.md:54,941`).
- `ndc` (NPI/NDC policy afforestation): density from `pm_carbon_density_secdforest_ac_uncalib` — **uncalibrated** natveg regrowth curve (`module_32.md:81,942`), because this is new-establishment land, not existing managed forest.
- `aff` (carbon-price-driven endogenous afforestation): density switch-dependent on `s32_aff_plantation` — either `pm_carbon_density_secdforest_ac_uncalib` (natveg curve) or `pm_carbon_density_plantation_ac_uncalib` (plantation curve), **both uncalibrated** (`module_32.md:943`).

**Key structural difference from Module 35**: Module 32 has no analogue of Module 35's natural-origin blending mechanism (§2) — there is no area-weighted mixing of a calibrated and uncalibrated curve within a single `type32` category. Instead, Module 32 achieves the calibrated-vs-uncalibrated distinction by **routing entire type categories** to one curve or the other outright: `plant` always gets the calibrated curve, `ndc` and `aff` always get uncalibrated curves. Module 35's secondary forest, by contrast, is a *single* land pool (no sub-typing by origin at the `land` level) that must represent both "existing/managed" and "natural-origin-from-succession" area simultaneously — hence the need for the per-age-class blended `p35_carbon_density_secdforest` introduced 2026-04-20 (`module_35.md:301-357`).

Module 32 also computes a **carbon-dioxide-removal (CDR) per-age-class increment**, `p32_cdr_ac(t,j,ac)` (`presolve.gms:71-72`, `module_32.md:119-125`):
```gams
p32_cdr_ac(t,j,ac)$(ord(ac)>1 AND (ord(ac)-1)<=s32_planning_horizon/5)
  = p32_carbon_density_ac(t,j,"aff",ac,"vegc") - p32_carbon_density_ac(t,j,"aff",ac-1,"vegc");
```
feeding `q32_cdr_aff` → `vm_cdr_aff(j,ac,"bgc")`, which Module 56 uses to reward endogenous afforestation prospectively over a 50-year planning horizon. Module 35 has **no equivalent forward-looking CDR-increment mechanism** — its carbon accounting is purely the current-timestep stock equations in §1; sequestration is realized only via the timestep-to-timestep stock difference in `q52_emis_co2_actual`, not projected ahead.

Both modules feed the same shared `vm_carbon_stock(j,land,ag_pools,stockType)` interface (Module 32 writing the `"forestry"` slice, Module 35 writing `"primforest"`/`"secdforest"`/`"other"`), and both read carbon-density parameters supplied by Module 52 rather than computing Chapman-Richards growth themselves.

## Limitations / what is NOT modeled

- Neither Module 35 nor Module 32 mechanistically models forest growth — carbon densities are pre-computed exogenous Chapman-Richards curves from Module 52, parameterized (not process-simulated) with region/climate-class-specific `k`/`m` and, since 2026-04-20, FRA-2025-target-calibrated `k`.
- Soil carbon (`soilc`) is entirely outside Module 35's age-class carbon logic — no age-class soil function exists anywhere in the model (Module 52, `module_52.md:920-923`).
- Module 35's carbon output is realization-specific to `pot_forest_may24`; no alternate realization exists to compare against for this module.
