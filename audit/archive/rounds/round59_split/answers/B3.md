## 1. The two balance equations

Module 50 runs **two parallel nitrogen balances** — cropland and pasture — each of the form `efficiency × inputs ≥ withdrawals`.

### Cropland balance — `q50_nr_bal_crp(i2)`

`modules/50_nr_soil_budget/macceff_aug22/equations.gms:14-16`

```gams
q50_nr_bal_crp(i2) ..
    vm_nr_eff(i2) * v50_nr_inputs(i2)
    =g= sum(kcr,v50_nr_withdrawals(i2,kcr));
```

Mathematically: `SNUpE(i) × N_inputs(i) ≥ N_withdrawals(i)`. This is an **inequality**, not a stock-conservation equality — the model is free to over-supply nitrogen (the excess becomes surplus), but cannot under-supply relative to crop N requirements.

### Pasture balance — `q50_nr_bal_pasture(i2)`

`modules/50_nr_soil_budget/macceff_aug22/equations.gms:55-59`

```gams
q50_nr_bal_pasture(i2) ..
    vm_nr_eff_pasture(i2) * v50_nr_inputs_pasture(i2)
    =g= v50_nr_withdrawals_pasture(i2);
```

Same structure, using the pasture-specific efficiency `vm_nr_eff_pasture` (Nitrogen Use Efficiency, NUE) instead of cropland SNUpE (Soil Nitrogen Uptake Efficiency).

## 2. Every inflow term

### 2.1 Cropland inputs — `q50_nr_inputs(i2)`

`modules/50_nr_soil_budget/macceff_aug22/equations.gms:22-32`

```gams
q50_nr_inputs(i2) ..
    v50_nr_inputs(i2) =e=
    vm_res_recycling(i2,"nr")
      + sum((cell(i2,j2),kcr,w), vm_area(j2,kcr,w) * f50_nr_fix_area(kcr))
      + sum(cell(i2,j2),vm_fallow(j2) * f50_nr_fix_area("tece"))
      + vm_manure_recycling(i2,"nr")
      + sum(kli, vm_manure(i2, kli, "stubble_grazing","nr"))
      + vm_nr_inorg_fert_reg(i2,"crop")
      + sum(cell(i2,j2),vm_nr_som_fertilizer(j2))
      + sum(ct,f50_nitrogen_balanceflow(ct,i2))
      + v50_nr_deposition(i2,"crop");
```

Eight terms, each with its own origin:

| # | Term | Meaning | Source module |
|---|------|---------|----------------|
| 1 | `vm_res_recycling(i2,"nr")` | Crop-residue N recycled to soil | **Module 18** |
| 2 | `sum((cell,kcr,w), vm_area(j2,kcr,w) × f50_nr_fix_area(kcr))` | Biological N fixation, area-based (legumes) | Module 30 (`vm_area`) × M50 input data |
| 3 | `sum(cell, vm_fallow(j2) × f50_nr_fix_area("tece"))` | N fixation on fallow land | Module 29 (`vm_fallow`) |
| 4 | `vm_manure_recycling(i2,"nr")` | Manure recycled to cropland from confinement AWMS | **Module 55** |
| 5 | `sum(kli, vm_manure(i2,kli,"stubble_grazing","nr"))` | Manure from animals grazing crop residues in the field | **Module 55** |
| 6 | `vm_nr_inorg_fert_reg(i2,"crop")` | Inorganic fertilizer — **the endogenous optimization variable** | Module 50 itself |
| 7 | `sum(cell, vm_nr_som_fertilizer(j2))` | Soil organic matter mining/mineralization | Module 59 |
| 8 | `sum(ct, f50_nitrogen_balanceflow(ct,i2))` | Exogenous correction term for unrealistic historical efficiencies | Input data |
| 9 | `v50_nr_deposition(i2,"crop")` | Atmospheric wet+dry deposition | Computed within M50 (see §2.3) |

(Doc numbers this "8 input sources" — deposition and balanceflow are separately itemized in the doc's prose but both appear as additive terms above; I count 9 additive terms in the literal formula, matching the equation as written.)

## 3. Every outflow term

### 3.1 Cropland withdrawals — `q50_nr_withdrawals(i2,kcr)`

`equations.gms:36-43`

```gams
q50_nr_withdrawals(i2,kcr) ..
    v50_nr_withdrawals(i2,kcr) =e=
    (1-sum(ct,f50_nr_fix_ndfa(ct,i2,kcr))) *
    (vm_prod_reg(i2,kcr) * fm_attributes("nr",kcr)
       + vm_res_biomass_ag(i2,kcr,"nr")
       + vm_res_biomass_bg(i2,kcr,"nr"))
    - vm_dem_seed(i2,kcr)  * fm_attributes("nr",kcr);
```

`N_withdrawal = (1 − NDFA) × [N_harvest + N_residue_AG + N_residue_BG] − N_seed`

- **NDFA** (`f50_nr_fix_ndfa`) = Nitrogen Derived From Atmosphere; for legumes 0.5–0.8, meaning that fraction of plant N need not come from soil at all — it's excluded from the withdrawal.
- **This is where crop-residue N enters as an outflow**: `vm_res_biomass_ag(i2,kcr,"nr")` and `vm_res_biomass_bg(i2,kcr,"nr")` — the total N content of above- and below-ground residue biomass — are **Module 18** outputs (`q18_prod_res_ag_reg`, `q18_prod_res_bg_reg`; `modules/18_residues/flexreg_apr16/equations.gms:14-19,24-28`) and are added into the withdrawal total, because the *whole plant* (harvest + residues) draws its N from the soil, not just the harvested fraction.
- Seed N is subtracted (`vm_dem_seed`, from Module 16) because seed brings N *into* the field at planting.

### 3.2 Pasture withdrawals — `q50_nr_withdrawals_pasture(i2)`

`equations.gms:83-85`

```gams
q50_nr_withdrawals_pasture(i2) ..
    v50_nr_withdrawals_pasture(i2) =e=
    vm_prod_reg(i2,"pasture") * fm_attributes("nr","pasture");
```

Simpler: just the N content of grazed grass production, no residue or NDFA term.

## 4. Where crop-residue N (Module 18) enters — both sides of the balance

Residue N is **not single-purpose** — it appears on both the input and withdrawal sides, which is the correct nutrient-cycle bookkeeping rather than double counting:

1. **As an outflow (withdrawal)**: `vm_res_biomass_ag` + `vm_res_biomass_bg` (total residue N contained in the plant biomass) are added into `q50_nr_withdrawals` (`equations.gms:40-41`) — because that N had to come from the soil in the first place, regardless of what happens to the residue afterward.
2. **As an inflow (input)**: `vm_res_recycling(i2,"nr")` — the fraction of residue N that Module 18 determines actually returns to soil after accounting for removal/burning/recycling fates — enters `q50_nr_inputs` (`equations.gms:24`).

`vm_res_recycling(i,"nr")` is itself computed in Module 18's `q18_res_recycling_nr(i2)` (`modules/18_residues/flexreg_apr16/equations.gms:90-96`):

```gams
q18_res_recycling_nr(i2) ..
    vm_res_recycling(i2,"nr") =e=
    sum(kcr, v18_res_ag_recycling(i2,kcr,"nr")
            + vm_res_ag_burn(i2,kcr,"nr")*(1-f18_res_combust_eff(kcr))
            + vm_res_biomass_bg(i2,kcr,"nr"));
```

Three N sources: (a) AG residue explicitly left on the field, (b) the fraction of burned-residue N that survives combustion and stays as ash, and (c) **all** belowground residue N (roots are never harvested in the default realization, so 100% of BG N returns to soil). Net effect: only the AG-removed fraction (feed, bioenergy, other off-field uses) leaves the residue-N cycle entirely; everything else cycles back as an M50 input.

## 5. Where animal-waste N (Module 55) enters — three separate terms

Module 55's manure output enters Module 50 at **three distinct points**, reflecting the AWMS feed-manure spatial coupling (confinement → cropland via storage, grazing → in-situ):

1. **Cropland input, confinement manure**: `vm_manure_recycling(i2,"nr")` (cropland `q50_nr_inputs`, term 4, `equations.gms:27`) — total manure N recycled from all 9 confinement systems, computed in Module 55's `q55_manure_recycling(i,npk)` (`modules/55_awms/ipcc2006_aug16/equations.gms:83-87`), already net of system-specific storage losses (`i55_manure_recycling_share`).
2. **Cropland input, stubble-grazing manure**: `sum(kli, vm_manure(i2,kli,"stubble_grazing","nr"))` (cropland `q50_nr_inputs`, term 5, `equations.gms:28`) — manure deposited in-field by animals grazing crop residues, from Module 55's `q55_bal_manure(i,kli,awms,npk)` (`equations.gms:68-71`) restricted to `awms="stubble_grazing"`.
3. **Pasture input, grazing manure**: `sum(kli, vm_manure(i2,kli,"grazing","nr"))` (pasture `q50_nr_inputs_pasture`, term 1, `equations.gms:77`) — manure deposited by animals grazing pasture, same `q55_bal_manure` equation restricted to `awms="grazing"`.

Module 55 computes `vm_manure` via mass balance (`equations.gms:68-71`): `Manure NPK = Feed NPK × (1 − im_slaughter_feed_share)`, i.e. feed nutrients not retained in animal biomass (meat/milk/eggs) become manure. There is no fourth `awms` category ("fuel" — manure collected as household cooking fuel) feeding Module 50: that manure is diverted out of the soil-nitrogen system entirely and does not appear in either `q50_nr_inputs` or `q50_nr_inputs_pasture`.

## 6. Nitrogen surplus — computed, but NOT literally passed to Module 51

### 6.1 The surplus equations

**Cropland** — `q50_nr_surplus(i2)`, `equations.gms:46-49`:
```gams
q50_nr_surplus(i2) ..
    v50_nr_surplus_cropland(i2) =e=
    v50_nr_inputs(i2) - sum(kcr, v50_nr_withdrawals(i2,kcr));
```

**Pasture** — `q50_nr_surplus_pasture(i2)`, `equations.gms:62-66`:
```gams
q50_nr_surplus_pasture(i2) ..
    v50_nr_surplus_pasture(i2) =e=
    v50_nr_inputs_pasture(i2) - v50_nr_withdrawals_pasture(i2);
```

Surplus = inputs minus withdrawals; this is the nitrogen the SNUpE/NUE constraint didn't require the crop to absorb, and conceptually represents the pool available for loss to the environment.

### 6.2 Important precision point: the surplus variable itself never crosses the module boundary

`v50_nr_surplus_cropland` and `v50_nr_surplus_pasture` both carry the **`v50_` prefix**, which in MAgPIE's naming convention denotes a **module-internal** variable (local to module 50), as opposed to the **`vm_` prefix** used for **interface variables** shared across modules — `reference/GAMS_MAgPIE_Patterns.md:207-208`: "`vm_` = Interface (shared between modules)... `v{NN}_` = Module-internal (local to module NN)". Consistent with this, Module 50's own "Provides To" table (`modules/module_50.md` §6.2) lists only three variables flowing to Module 51: `vm_nr_eff`, `vm_nr_eff_pasture`, and `vm_nr_inorg_fert_reg`. The surplus variables are **not** in that table and are not read anywhere in Module 51's equations per `modules/module_51.md` (its "Inputs (from other modules)" section, lines 224-245, lists no `v50_nr_surplus_*` term).

So the literal mechanism is **not** "Module 50 computes a surplus number and Module 51 consumes it." Instead:

- Module 51 **independently reconstructs a source-specific surplus fraction for each nitrogen input category** using the same `(1 − efficiency)` logic, applied separately to manure, fertilizer, residues, and SOM — because it needs source-level attribution (different `i51_ef_n_soil` emission factors per source) that a single aggregated `v50_nr_surplus_cropland` number would not carry.
- The link variable is **`vm_nr_eff(i)`** / **`vm_nr_eff_pasture(i)`** — the *efficiency*, not the surplus. Since `vm_nr_eff` is fixed in Module 50's `presolve.gms:76-77` before the solve (`vm_nr_eff.fx(i) = 1 - (1-i50_nr_eff_bau(t,i)) * (1 - i50_maccs_mitigation_transf(t,i))`), Module 51 treats it as a known parameter at solve time.

### 6.3 The rescaling formula (Module 51, NUE-based rescaling)

For each direct-emission source, `modules/51_nitrogen/rescaled_jan21/equations.gms`:

**Manure on cropland**, `q51_emissions_man_crop` (`equations.gms:22-27`):
```gams
vm_emissions_reg(i2,"man_crop",n_pollutants_direct) =e=
vm_manure_recycling(i2,"nr") / (1-s51_snupe_base) * (1-vm_nr_eff(i2))
* sum(ct, i51_ef_n_soil(ct,i2,n_pollutants_direct,"man_crop"))
```

**Inorganic fertilizer** (crop + pasture), `q51_emissions_inorg_fert` (`equations.gms:30-39`):
```gams
vm_emissions_reg(i2,"inorg_fert",n_pollutants_direct) =e=
vm_nr_inorg_fert_reg(i2,"crop") / (1-s51_snupe_base) * (1-vm_nr_eff(i2))
   * sum(ct, i51_ef_n_soil(ct,i2,n_pollutants_direct,"inorg_fert"))
+ vm_nr_inorg_fert_reg(i2,"past") / (1-s51_nue_pasture_base) * (1-vm_nr_eff_pasture(i2))
   * sum(ct, i51_ef_n_soil(ct,i2,n_pollutants_direct,"inorg_fert"))
```

**Crop residues**, `q51_emissions_resid` (`equations.gms:42-46`):
```gams
vm_emissions_reg(i2,"resid",n_pollutants_direct) =e=
vm_res_recycling(i2,"nr") * sum(ct, i51_ef_n_soil(ct,i2,n_pollutants_direct,"resid"))
/ (1-s51_snupe_base) * (1-vm_nr_eff(i2))
```

**SOM loss**, `q51_emissions_som` (`equations.gms:55-59`):
```gams
vm_emissions_reg(i2,"som",n_pollutants_direct) =e=
sum(cell(i2,j2),vm_nr_som(j2)) * sum(ct, i51_ef_n_soil(ct,i2,n_pollutants_direct,"som"))
/ (1-s51_snupe_base) * (1-vm_nr_eff(i2))
```

**Manure on pasture**, `q51_emissionbal_man_past` (`equations.gms:74-80`):
```gams
vm_emissions_reg(i2,"man_past",n_pollutants_direct) =e=
sum((awms_prp,kli), vm_manure(i2,kli,awms_prp,"nr") * f51_ef3_prp(i2,n_pollutants_direct,kli))
/ (1-s51_nue_pasture_base) * (1-vm_nr_eff_pasture(i2))
```

**Rescaling logic** (all five equations above): `s51_snupe_base = 0.5` (`input.gms:8`) is the SNUpE the IPCC 2006 emission factors implicitly assume. Dividing the raw N-input flow by `(1 − 0.5)` and multiplying by `(1 − vm_nr_eff)` rescales IPCC's assumed-50%-loss-fraction emissions to the *actual* MAgPIE-computed efficiency. Algebraically this is exactly the "loss fraction" (1 minus efficiency) applied to that source's own N-input flow — i.e., a **per-source surplus reconstruction**, not a readout of Module 50's single `v50_nr_surplus_cropland` aggregate. Two direct-emission pathways are explicitly **not** rescaled this way: residue burning (`q51_emissions_resid_burn`, combustion is independent of soil uptake efficiency) and confined-manure AWMS emissions (`q51_emissionbal_awms`, uses MACC mitigation `im_maccs_mitigation` instead).

**Indirect N₂O** (`q51_emissions_indirect_n2o`, `equations.gms:83-89`) then derives secondary N₂O from the volatilized NH₃/NOₓ and leached NO₃⁻ computed by the direct-emission equations above (IPCC EF4 = 0.01, EF5 = 0.0075), closing the emissions accounting that Module 56 (GHG Policy) subsequently prices.

### 6.4 Net summary of the M50 → M51 handoff

| What crosses the module boundary | What does NOT |
|---|---|
| `vm_nr_eff(i)`, `vm_nr_eff_pasture(i)` — fixed SNUpE/NUE (`presolve.gms:76-77`) | `v50_nr_surplus_cropland`, `v50_nr_surplus_pasture` (internal `v50_` variables, never read by M51) |
| `vm_nr_inorg_fert_reg(i,"crop"/"past")` — raw fertilizer flow | Any single "total surplus" figure |
| (indirectly, via M18/M55 which feed both M50 and M51 directly) `vm_res_recycling`, `vm_manure_recycling`, `vm_manure` | |

Module 51 is therefore best described as **recomputing** a source-disaggregated version of Module 50's surplus concept, keyed to the same efficiency variable, rather than consuming Module 50's aggregated surplus output directly. `cross_module/nitrogen_food_balance.md:59-66` corroborates this framing: "Module 50 calculates nitrogen surplus (`v50_nr_surplus_cropland`)... Emissions are calculated via NUE-rescaling, NOT simple emission factors: `emission = source / (1 - s51_snupe_base) * (1 - vm_nr_eff) * ef`."

## 7. What is explicitly NOT modeled here (per module docs)

- Module 50 does **not** track a soil mineral-N stock across timesteps — every variable is a period flow; the surplus is an unconstrained residual, not a pool that could deplete or accumulate (`cross_module/nitrogen_food_balance.md:88,96`; `modules/module_50.md` §13.10). There is consequently no N-limitation feedback on yields.
- `vm_nr_eff` / `vm_nr_eff_pasture` are declared as `positive variables` but are **fixed** by `presolve.gms:76-77` — not solver-optimized. Only `vm_nr_inorg_fert_reg` is truly endogenous.
- The nitrogen balance is explicitly **not a conservation law** in MAgPIE's own classification (`cross_module/nitrogen_food_balance.md:283,297-299`): inputs (fertilizer) and outputs (emissions) cross the model boundary freely; only mass-balance *accounting* is enforced, not physical closure.
- Module 51 has **no feedback loop** back into Module 50 or livestock/crop decisions — emissions are a terminal calculation consumed only by Module 56 (pricing) and Module 57 (MACC cost integral) (`modules/module_51.md` §"Critical Hub Status").
