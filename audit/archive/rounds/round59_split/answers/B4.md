## 1. How inorganic fertilizer demand is determined (Module 50)

Inorganic fertilizer demand is **not calculated by a direct formula** — it is the value `vm_nr_inorg_fert_reg(i,land_ag)` that the solver must choose so that two nitrogen mass-balance inequality constraints hold, at minimum cost.

**Cropland balance** — `modules/50_nr_soil_budget/macceff_aug22/equations.gms:14-16` (`modules/module_50.md:32-38`):

```gams
q50_nr_bal_crp(i2) ..
    vm_nr_eff(i2) * v50_nr_inputs(i2)
    =g= sum(kcr,v50_nr_withdrawals(i2,kcr));
```

i.e. `SNUpE(i) × N_inputs(i) ≥ N_withdrawals(i)`. `v50_nr_inputs(i2)` is itself a sum of eight terms (`equations.gms:22-32`, `modules/module_50.md:83-97`), one of which is the fertilizer variable:

```gams
q50_nr_inputs(i2) ..
    v50_nr_inputs(i2) =e=
    vm_res_recycling(i2,"nr")
      + sum((cell(i2,j2),kcr,w), vm_area(j2,kcr,w) * f50_nr_fix_area(kcr))
      + sum(cell(i2,j2),vm_fallow(j2) * f50_nr_fix_area("tece"))
      + vm_manure_recycling(i2,"nr")
      + sum(kli, vm_manure(i2, kli, "stubble_grazing","nr"))
      + vm_nr_inorg_fert_reg(i2,"crop")
      + sum(cell(i2,j2),vm_nr_som_fertilizer(j2))
      + sum(ct,f50_nitrogen_balanceflow(ct,i2))
      + v50_nr_deposition(i2,"crop");
```

The pasture analogue is `q50_nr_bal_pasture` / `q50_nr_inputs_pasture` (`equations.gms:55-59, 74-80`; `modules/module_50.md:60-68, 111-121`), with `vm_nr_inorg_fert_reg(i2,"past")` as one of four input terms.

Because `vm_nr_eff(i2)` (SNUpE, cropland) and `vm_nr_eff_pasture(i2)` are **fixed** each timestep in presolve (`presolve.gms:76-77`, `modules/module_50.md:298-301, 399-417`) — computed from BAU scenario data plus a MACC-curve transform, not optimized — the only free lever left to satisfy the inequality is fertilizer application. The solver picks the minimum `vm_nr_inorg_fert_reg` that closes the gap between fixed non-fertilizer inputs and required withdrawals at the fixed efficiency:

```
fertilizer(i) = N_withdrawal(i)/SNUpE(i) − N_inputs_non_fert(i)
```

(`modules/module_50.md:794-807`, illustrative worked numbers only). So: **fertilizer demand is endogenous** — a decision variable jointly determined with all other land-use/production choices to minimize total system cost subject to the nitrogen balance (and every other) constraint — while the **efficiency it is applied at is exogenous** (scenario-driven).

## 2. Fertilizer cost, the carrying variable, and where it is declared

**Cost equation** — `modules/50_nr_soil_budget/macceff_aug22/equations.gms:94-97` (`modules/module_50.md:219-229`):

```gams
q50_nr_cost_fert(i2) ..
    vm_nr_inorg_fert_costs(i2) =e=
    sum(land_ag,vm_nr_inorg_fert_reg(i2,land_ag)) * s50_fertilizer_costs;
```

- **Carrying variable**: `vm_nr_inorg_fert_costs(i)` — regional inorganic-N fertilizer cost, units mio. USD17MER/yr.
- **Formula**: total fertilizer quantity applied (summed over `land_ag` = crop + pasture) × a flat per-tonne price.
- **Price parameter**: `s50_fertilizer_costs = 738` USD17MER per tN, set in `modules/50_nr_soil_budget/macceff_aug22/input.gms:29` (`modules/module_50.md:227, 327`).

**Declaring module**: `vm_nr_inorg_fert_costs` is DECLARED in **Module 50** — `modules/50_nr_soil_budget/macceff_aug22/declarations.gms:11` (`modules/module_50.md:287`, table row: "vm_nr_inorg_fert_costs | positive | (i) | mio USD17MER/yr | [0,∞) | declarations.gms:11"). It is POPULATED by `q50_nr_cost_fert` in the same module, and READ (consumed) downstream by Module 11.

🟢 Verified against `modules/module_50.md` §5.3, §7, §9 (module doc; DECLARED-vs-POPULATED-vs-READ roles distinguished per `agent/helpers/verifiers.md` MANDATE 18).

## 3. How it reaches Module 11's objective

Module 11 (`default` realization, the only one) consumes `vm_nr_inorg_fert_costs` **directly** — one hop, no intermediate routing module — as a term in its regional cost-aggregation equation:

`modules/11_costs/default/equations.gms:15-47`, specifically line 24 (`modules/module_11.md:86-119`):

```gams
q11_cost_reg(i2) .. v11_cost_reg(i2) =e= sum(factors,vm_cost_prod_crop(i2,factors))
                   + sum(kres,vm_cost_prod_kres(i2,kres))
                   + vm_cost_prod_past(i2)
                   + vm_cost_prod_fish(i2)
                   + sum(factors,vm_cost_prod_livst(i2,factors))
                   + sum((cell(i2,j2),land), vm_cost_landcon(j2,land))
                   + sum((cell(i2,j2),k), vm_cost_transp(j2,k))
                   + vm_tech_cost(i2)
                   + vm_rotation_penalty(i2)
                   + vm_nr_inorg_fert_costs(i2)        <-- Module 50's term
                   + vm_p_fert_costs(i2)
                   ... [21 more terms; equations.gms:15-47]
;
```

`v11_cost_reg(i2)` (regional total cost, one of 32 summed terms — 31 positive + the single negative `vm_reward_cdr_aff` term) is then aggregated to the global objective by `q11_cost_glo` (`equations.gms:10`, `modules/module_11.md:47-77`):

```gams
q11_cost_glo .. vm_cost_glo =e= sum(i2, v11_cost_reg(i2));
```

`vm_cost_glo` (mio. USD17MER/yr) is the variable the GAMS solver **minimizes** — MAgPIE's entire objective function (`modules/module_11.md:16-18, 517-523`). So the path is:

```
vm_nr_inorg_fert_reg (M50, endogenous decision var)
  → q50_nr_cost_fert → vm_nr_inorg_fert_costs (M50, declared M50 declarations.gms:11)
    → q11_cost_reg (M11 equations.gms:24) → v11_cost_reg
      → q11_cost_glo (M11 equations.gms:10) → vm_cost_glo  [OBJECTIVE, minimized]
```

Per Module 11's docs, this is a **direct, one-hop** consumer relationship (M50 → M11), the same pattern the docs flag for `vm_p_fert_costs` ("routed M54 → M38 → M11 but wired DIRECTLY into M11" is the cautionary example in `agent/helpers/verifiers.md` MANDATE 18 — the nitrogen term follows the same direct-wiring pattern, confirmed by its explicit appearance at `equations.gms:24` inside `q11_cost_reg` itself, not via any pass-through module).

## 4. Exogenous or endogenous fertilizer price?

**The price is exogenous; the quantity is endogenous.**

- `s50_fertilizer_costs = 738` USD17MER/tN is a **fixed scalar**, set once in `modules/50_nr_soil_budget/macceff_aug22/input.gms:29`. It does not appear on the left-hand side of any equation in Module 50 — nothing in the model computes or adjusts it. It is uniform across all regions and constant over all time steps under the default configuration.
- Explicitly documented as a limitation: "Does NOT vary fertilizer costs regionally or temporally — `s50_fertilizer_costs = 738` USD/tN globally, constant over time (`input.gms:29`). No price dynamics or regional fertilizer markets." (`modules/module_50.md:511-513`)
- Module 11 itself is a pure aggregator with "No Dynamic Pricing or Markets... All costs are exogenous or calculated by source modules before reaching Module 11" (`modules/module_11.md:696-700`), consistent with M50 supplying a fixed unit price.
- By contrast, `vm_nr_inorg_fert_reg` (the tonnage) **is** an endogenous optimization variable, re-derived every timestep from the nitrogen mass balance jointly with all other model decisions (see §1 above).

So: `Cost = (endogenous quantity) × (exogenous fixed price)`. There is no supply/demand feedback, scarcity pricing, or MACC-driven adjustment of the *price* — only of the *efficiency* (SNUpE/NUE) at which that price's associated fertilizer is used, which indirectly affects how much quantity is demanded, not the per-tonne price itself.
