## Module 40 (Transport): Default Realization and Transport Cost Computation

Source: `modules/module_40.md`

---

### Default Realization

The default realization is **`gtap_nov12`**, confirmed in `config/default.cfg`:
`cfg$gms$transport <- "gtap_nov12"`.

The alternative realization `off` sets `vm_cost_transp.fx(j,k) = 0` in `off/presolve.gms:9`, eliminating all transport costs (counterfactual/sensitivity use only).

---

### Cost Variable

**`vm_cost_transp(j,k)`** — Transportation costs, declared at `gtap_nov12/declarations.gms:13`.

- Dimensions: `j` (spatial cluster cells, ~200 globally) x `k` (commodity types, kall set)
- Units: Million USD17MER per year

---

### Equation

There is exactly one equation in the `gtap_nov12` realization:

**`q40_cost_transport`** — located at `modules/40_transport/gtap_nov12/equations.gms:11-13`

```gams
q40_cost_transport(j2,k) ..
  vm_cost_transp(j2,k) =e= vm_prod(j2,k)*f40_distance(j2)
                           * f40_transport_costs(k);
```

This is a single multiplicative equality: transport cost per cell-commodity = production x distance x commodity cost factor. The relationship is perfectly linear in all three terms.

---

### Inputs

#### Production from Module 17

**`vm_prod(j2,k)`** — Cell-level production by commodity, provided by Module 17 (Production).

- Usage in equation: `equations.gms:12`
- Units: Million tons dry matter per year (tDM/yr)
- Coupling: Simultaneous — Module 17 and Module 40 equations are solved together in each optimization timestep. Transport costs scale linearly with production volume, creating a negative feedback that can limit concentration of production in high-yield but remote cells.

#### Transport Distance

**`f40_distance(j2)`** — Travel time from each cell centroid to the nearest urban center with >=50,000 inhabitants.

- Source data: Nelson et al. 2008, EC JRC global travel time map (30 arc-second resolution, circa year 2000)
- Input file: `modules/40_transport/input/transport_distance.cs2`
- Declaration: `gtap_nov12/input.gms:15-21`
- Units: Minutes
- Aggregation: 30 arc-second (~1 km²) data area-weighted to MAgPIE cluster cells (~0.5 degrees)
- Critical limitation: `f40_distance` is **static throughout the simulation** — infrastructure improvements, urbanization, and new roads are not modeled. Travel times are frozen at circa-2000 levels for all timesteps (`realization.gms:13-16` states this explicitly).

#### Commodity Cost Factor

**`f40_transport_costs(k)`** — Commodity-specific transport cost factor calibrated to GTAP 7 data.

- Source data: Narayanan & Walmsley 2008, GTAP 7 database (Purdue University)
- Input file: `modules/40_transport/gtap_nov12/input/f40_transport_costs.csv`
- Declaration: `gtap_nov12/input.gms:23-28`
- Units: USD17MER per ton dry matter per minute
- Calibration: Iterative process (typically 2-3 rounds) matching MAgPIE's optimized spatial production patterns against GTAP 7 aggregate transport costs scaled from 2004 to 1995. The calibration procedure is documented at `equations.gms:49-68`.
- Pasture exception: **`s40_pasture_transport_costs = 0`** (`input.gms:10`, set at line 30) — grass is consumed in situ by grazing animals, not transported, so pasture incurs zero transport cost regardless of remoteness.

---

### Unit Arithmetic

[Million tDM/yr] x [minutes] x [USD17/tDM/min] = [Million USD17/yr]

The factor of 1,000,000 from tDM -> Million tDM cancels with USD -> Million USD, yielding the correct units for `vm_cost_transp`.

---

### How the Cost Reaches Module 11 (Objective Function)

`vm_cost_transp(j,k)` is the sole output variable from Module 40. Module 11 (Costs) receives it and performs regional aggregation:

- Aggregation pattern (documented in `module_40.md` Interface Variables section): `sum(j2, vm_cost_transp(j2,k))` summed by region `i`
- The aggregated transport costs are added into the regional cost totals that feed into Module 11's objective function
- Effect on optimization: Cells with high `f40_distance` (remote) are penalized — producing in remote cells costs more, so the optimizer favors accessible cells (ceteris paribus). This implements a von Thunen-style land rent gradient where accessibility is one component of cell attractiveness.

The output reporting parameters are assigned in `gtap_nov12/postsolve.gms:10-19`:
- `ov_cost_transp(t,j,k,type)`: Variable values over time
- `oq40_cost_transport(t,j,k,type)`: Equation marginals over time

These are exported to GDX and read by R post-processing.
