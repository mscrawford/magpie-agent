## How vm_carbon_stock is computed and priced

### 1. Declaration and ownership

`vm_carbon_stock(j,land,c_pools,stockType)` is declared in **Module 56** (`declarations.gms:34`). It has dimensions:
- `j` — simulation cells
- `land` — land types (crop, past, primforest, secdforest, urban, other, and plantation subtypes)
- `c_pools` — carbon pools (vegc, litc, soilc)
- `stockType` — stock type (e.g., "actual", "actualNoAcEst")

Units: mio. tC.

Module 52 does NOT compute `vm_carbon_stock` — it *reads* it. The variable is populated by the **land modules** during optimization (see Section 2 below). Module 52 uses the result to calculate CO2 emissions.

---

### 2. Modules that POPULATE vm_carbon_stock

Six modules write to `vm_carbon_stock` via their equations during the solve:

| Module | Land types covered | Pools covered |
|--------|--------------------|---------------|
| **Module 29 (Cropland)** | crop | vegc, litc (folds in `vm_carbon_stock_croparea` from Module 30) |
| **Module 31 (Pasture)** | past | vegc, litc, soilc |
| **Module 32 (Forestry)** | plantation/forestry land | vegc, litc, soilc |
| **Module 34 (Urban)** | urban | fixed to 0 |
| **Module 35 (Natural Vegetation)** | primforest, secdforest, other | vegc, litc, soilc |
| **Module 59 (SOM)** | all land types | soilc pool specifically |

Additional notes:
- Module 30 (Croparea) populates the *separate* variable `vm_carbon_stock_croparea`, which Module 29 then folds into its `vm_carbon_stock` contribution. Module 30 does not directly write to `vm_carbon_stock`.
- Module 58 (Peatland) does NOT populate `vm_carbon_stock` — peatland emissions take their own pathway.

Sources: module_52.md Interface Variables section; module_56.md Section 4.1 and 12.4.

### 3. How Module 52 uses vm_carbon_stock: equation q52_emis_co2_actual

Module 52's single optimization equation computes CO2 emissions from land-use change as the difference between the previous and current carbon stock.

**Equation**: `q52_emis_co2_actual` (`modules/52_carbon/normal_dec17/equations.gms:16-19`)

```gams
q52_emis_co2_actual(i2,emis_oneoff) ..
  vm_emissions_reg(i2,emis_oneoff,"co2_c") =e=
    sum((cell(i2,j2),emis_land(emis_oneoff,land,c_pools)),
      (pcm_carbon_stock(j2,land,c_pools,"actual") - vm_carbon_stock(j2,land,c_pools,"actual"))
      / m_timestep_length);
```

**Components**:
- `pcm_carbon_stock(j,land,c_pools,"actual")` — previous timestep's carbon stock (mio. tC), stored by Module 56 at the end of each period; declared in Module 56 `declarations.gms`
- `vm_carbon_stock(j,land,c_pools,"actual")` — current timestep carbon stock being optimized (mio. tC)
- `m_timestep_length` — timestep length in years (macro defined in `core/macros.gms:51`)
- `emis_land(emis_oneoff,land,c_pools)` — set mapping emission sources to land-pool combinations (e.g., `crop_vegc -> crop, vegc`)
- `vm_emissions_reg(i2,emis_oneoff,"co2_c")` — output: annualized CO2-C emissions by region and source (Tg C/yr)

**Sign convention**: When `pcm_carbon_stock > vm_carbon_stock` (carbon loss), emissions are positive (source). When `pcm_carbon_stock < vm_carbon_stock` (carbon gain), emissions are negative (sequestration).

**Emission sources covered** (`emis_oneoff` set): crop_vegc, crop_litc, crop_soilc, past_vegc, past_litc, past_soilc, forestry_vegc, forestry_litc, forestry_soilc, primforest_vegc, primforest_litc, primforest_soilc, secdforest_vegc, secdforest_litc, secdforest_soilc, urban_vegc, urban_litc, urban_soilc, other_vegc, other_litc, other_soilc.

Source: module_52.md, Section 3 and Key Equations section (equations.gms:16-19 cited).

### 4. Where vm_carbon_stock enters GHG policy cost in Module 56

Module 56 (realization `price_aug22`) uses `vm_carbon_stock` via a **direct pathway that bypasses `vm_emissions_reg`** for the CO2 pricing calculation.

#### 4a. Equation q56_emis_pricing_co2 (equations.gms:19-22)

```gams
q56_emis_pricing_co2(i2,emis_oneoff) ..
  v56_emis_pricing(i2,emis_oneoff,"co2_c") =e=
    sum((cell(i2,j2),emis_land(emis_oneoff,land,c_pools)),
      (pcm_carbon_stock(j2,land,c_pools,"actual") - vm_carbon_stock(j2,land,c_pools,"%c56_carbon_stock_pricing%"))
      / m_timestep_length);
```

This is the **direct carbon stock pathway**. For CO2 pricing purposes, Module 56 re-derives the carbon stock change directly from `vm_carbon_stock` (not from `vm_emissions_reg`). The `stockType` used here is controlled by the switch `c56_carbon_stock_pricing`:
- `"actual"` — all carbon stock changes are priced
- `"actualNoAcEst"` (default) — excludes afforestation establishment age class from pricing, preventing double-counting with the CDR reward mechanism

Note: `q52_emis_co2_actual` (Module 52) produces `vm_emissions_reg(...,"co2_c")` which IS used for reporting, but for pricing, Module 56 goes directly to `vm_carbon_stock` via `q56_emis_pricing_co2`. The two equations run in parallel and are conceptually consistent but structurally independent.

Contrast with annual emissions (CH4, N2O): those flow through `vm_emissions_reg` via `q56_emis_pricing` (`equations.gms:15-17`), not through `vm_carbon_stock` directly.

Source: module_56.md Section 2.2 (equations.gms:19-22, Architectural Note).

#### 4b. Equation q56_emission_cost_oneoff (equations.gms:45-52)

The priced CO2 emissions from one-off events are then costed using an annuity factor:

```gams
q56_emission_cost_oneoff(i2,emis_oneoff) ..
  v56_emission_cost(i2,emis_oneoff) =e=
    sum(pollutants,
      v56_emis_pricing(i2,emis_oneoff,pollutants)
      * m_timestep_length
      * sum(ct,
          im_pollutant_prices(ct,i2,pollutants,emis_oneoff)
          * pm_interest(ct,i2)/(1+pm_interest(ct,i2))));
```

The annuity factor `r/(1+r)` converts the one-time emission event into an equivalent perpetual annual cost, so that deforestation (one-time emission) is fairly compared against recurring annual emission sources.

Source: module_56.md Section 2.4 (equations.gms:45-52).

#### 4c. Equation q56_emission_costs: aggregate to vm_emission_costs (equations.gms:56-58)

```gams
q56_emission_costs(i2) ..
  vm_emission_costs(i2) =e=
    sum(emis_source, v56_emission_cost(i2,emis_source));
```

`vm_emission_costs(i)` is the final aggregate (mio. USD17MER/yr) that enters Module 11 (Costs) and the objective function. Minimizing this cost creates the land-use response to the carbon price.

Source: module_56.md Section 2.5 (equations.gms:56-58).

### 5. Carbon density inputs that Module 52 provides to support vm_carbon_stock calculations

While Module 52 does not write `vm_carbon_stock`, it provides the age-class-specific carbon density parameters that the land modules use to populate it:

- `pm_carbon_density_secdforest_ac(t_all,j,ac,ag_pools)` — secondary forest carbon density by age class (tC/ha), calibrated via Chapman-Richards equation (start.gms:28,31) with optional bisection calibration to FAO FRA 2025 targets in preloop.gms when `s52_growingstock_calib = 1`. Consumed by Module 35 (`modules/35_natveg/pot_forest_may24/presolve.gms:248`).
- `pm_carbon_density_plantation_ac(t_all,j,ac,ag_pools)` — plantation carbon density by age class (tC/ha), similarly calibrated. Consumed by Module 32 (`modules/32_forestry/dynamic_may24/presolve.gms:65`).
- `pm_carbon_density_other_ac(t_all,j,ac,ag_pools)` — other land carbon density by age class (not calibrated). Consumed by Module 35.
- `fm_carbon_density(t_all,j,land,c_pools)` — base LPJmL carbon densities for non-age-class land types. Consumed by Modules 14, 29, 30, 31, 35, 56, and 59 for carbon stock calculations.

Also note: Module 52's preloop.gms reads `fm_carbon_density` directly at `modules/56_ghg_policy/price_aug22/preloop.gms:10` for initializing `pcm_carbon_stock` at simulation start (module_52.md Interface section, fm_carbon_density consumer list).

### 6. Summary of the full chain

```
Land modules (29, 31, 32, 34, 35, 59)
    ↓ populate during optimization
vm_carbon_stock(j,land,c_pools,stockType)   [declared in M56, dimensions: j × land × c_pools × stockType]
    ↓ read by M52 equation
q52_emis_co2_actual (equations.gms:16-19)
    ↓ produces
vm_emissions_reg(i,emis_oneoff,"co2_c")      [used for reporting / non-CO2 gases priced via q56_emis_pricing]

SIMULTANEOUSLY, M56 reads vm_carbon_stock directly:
vm_carbon_stock + pcm_carbon_stock
    ↓ read by M56 equation q56_emis_pricing_co2 (equations.gms:19-22)
v56_emis_pricing(i,emis_oneoff,"co2_c")
    ↓ multiplied by price + annuity factor in q56_emission_cost_oneoff (equations.gms:45-52)
v56_emission_cost(i,emis_oneoff)
    ↓ aggregated in q56_emission_costs (equations.gms:56-58)
vm_emission_costs(i)
    ↓ enters Module 11 (Costs) objective function
```

Key architectural point: for CO2 pricing, Module 56 reads `vm_carbon_stock` directly (bypassing `vm_emissions_reg`) via the dedicated equation `q56_emis_pricing_co2`. This is a deliberate design to allow `c56_carbon_stock_pricing` to control which stockType is priced independently of how emissions are reported.
