QUESTION: In MAgPIE's default configuration, which realization of module 18 (residues) is active, and how does it compute and allocate crop / above-ground residues? Name the key variables and equations (q18_*), and explain how residues link to production (module 17) and to demand/recycling (modules 16 / 50). Cite exact names and file:line.

---

## Active Realization

The default realization is **`flexreg_apr16`**, confirmed in `config/default.cfg:622` (`cfg$gms$residues <- "flexreg_apr16"`). It has **9 equations** all operating at the regional level (i), except one cluster-sum constraint. The alternative realizations are `flexcluster_jul23` (12 equations, cluster-level AG accounting) and `off` (no residue accounting).

Source: `module_18.md`, verified against `modules/18_residues/flexreg_apr16/*.gms`.

## Residue Classification

Crop residues are classified into three **homogeneous residue groups** (`kres`) via the `kres_kcr(kres,kcr)` mapping (`flexreg_apr16/sets.gms:28-34`):

- `res_cereals` -> tece, maiz, trce, rice_pro
- `res_fibrous` -> soybean, rapeseed, groundnut, puls_pro, sugr_beet, sugr_cane, cottn_pro
- `res_nonfibrous` -> potato, cassav_sp, others

Five crop types have removal fixed to zero (`flexreg_apr16/presolve.gms:8`): `v18_res_ag_removal.fx(i,nonused18,attributes) = 0` for `sunflower, oilpalm, foddr, begr, betr`.

## Computation of Aboveground Residue Biomass

### q18_prod_res_ag_reg (`flexreg_apr16/equations.gms:14-19`)

```gams
q18_prod_res_ag_reg(i2,kcr,attributes) ..
    vm_res_biomass_ag(i2,kcr,attributes)
    =e=
    (sum((cell(i2,j2),w), vm_area(j2,kcr,w)) * sum(ct,f18_multicropping(ct,i2)) * f18_cgf("intercept",kcr)
    + vm_prod_reg(i2,kcr) * f18_cgf("slope",kcr))
    * f18_attributes_residue_ag(attributes,kcr);
```

Regional AG residue biomass (`vm_res_biomass_ag(i,kcr,attributes)`) is computed from two terms using IPCC 2006 crop growth functions (CGFs):
- **Intercept term**: harvested area (`vm_area(j,kcr,w)` from module 30) x multicropping factor (`f18_multicropping`) x area-based intercept (`f18_cgf("intercept",kcr)`)
- **Slope term**: regional crop production (`vm_prod_reg(i,kcr)` from module 17) x residue-to-product ratio (`f18_cgf("slope",kcr)`)

The result is multiplied by `f18_attributes_residue_ag(attributes,kcr)` to convert dry-matter biomass into per-attribute units (dm, nr, c, gj, p, k, wm).

**Link to module 17**: `vm_prod_reg(i,kcr)` (the kcr-slice of the shared `vm_prod_reg` symbol) is the direct input from module 17's production equations. This is a one-way, non-circular dependency.

### q18_prod_res_bg_reg (`flexreg_apr16/equations.gms:24-28`)

Belowground residue biomass:

```gams
vm_res_biomass_bg(i2,kcr,dm_nr) =e=
    (vm_prod_reg(i2,kcr) + vm_res_biomass_ag(i2,kcr,"dm")) * f18_cgf("bg_to_ag",kcr)
    * f18_attributes_residue_bg(dm_nr,kcr);
```

Total aboveground biomass (harvested production + AG residues) times the BG-to-AG ratio gives belowground biomass. BG residues always remain in soil.

## Allocation of Aboveground Residues (Field Balance)

### q18_res_field_balance (`flexreg_apr16/equations.gms:38-43`)

```gams
q18_res_field_balance(i2,kcr,attributes) ..
    vm_res_biomass_ag(i2,kcr,attributes)
    =e=
    v18_res_ag_removal(i2,kcr,attributes)
    + vm_res_ag_burn(i2,kcr,attributes)
    + v18_res_ag_recycling(i2,kcr,attributes);
```

Every unit of AG biomass must go to exactly one of three fates:
1. **Removal** (`v18_res_ag_removal`) - harvested for feed, bioenergy, or other uses
2. **Burning** (`vm_res_ag_burn`) - burned on field
3. **Recycling** (`v18_res_ag_recycling`) - left on field to decay

All three fate variables are at regional level (i) in `flexreg_apr16`. There is no cluster-level field balance in the default realization; this is the key spatial limitation (see Limitation 4 in `module_18.md`).

### q18_res_field_burn (`flexreg_apr16/equations.gms:52-57`)

The burn allocation is exogenously determined:

```gams
vm_res_ag_burn(i2,kcr,attributes) =e=
    sum(ct, im_development_state(ct,i2) * i18_res_use_burn(ct,"high_income",kcr)
    + (1-im_development_state(ct,i2)) * i18_res_use_burn(ct,"low_income",kcr))
    * vm_res_biomass_ag(i2,kcr,attributes);
```

Burn share is a development-state-weighted average of historical high-income (15%) and low-income (25%) burn rates (Smil 1999). The development indicator `im_development_state(t,i)` comes from module 09. Under the default `c18_burn_scen = "phaseout"`, burn shares decline toward 2050 (`flexreg_apr16/preloop.gms:9`).

## Translation to Homogeneous Residue Groups

### q18_translate (`flexreg_apr16/equations.gms:70-73`)

```gams
q18_translate(i2,kres,attributes) ..
    sum(kres_kcr(kres,kcr), v18_res_ag_removal(i2,kcr,attributes))
    =e=
    vm_prod_reg(i2,kres) * fm_attributes(attributes,kres);
```

This equation translates crop-specific residue removals (heterogeneous nutrient content) into the three homogeneous residue-group production quantities `vm_prod_reg(i,kres)`. The mass balance is preserved per attribute.

**Link to module 16 (Demand) and module 21 (Trade)**: `vm_prod_reg(i,kres)` (the kres-slice) is the production supply quantity that feeds into the demand and trade system. Module 16 sees residue production as part of the supply-demand balance for feed and bioenergy commodities. Module 21 (Trade) uses the same variable for inter-regional trade accounting.

### q18_prod_res_reg (`flexreg_apr16/equations.gms:78-81`)

```gams
q18_prod_res_reg(i2,kres) ..
    sum(cell(i2,j2), v18_prod_res(j2,kres)) =e= vm_prod_reg(i2,kres);
```

Distributes regional residue production across clusters for spatial reporting. The cluster-level variable `v18_prod_res(j,kres)` is flexible (unconstrained at cluster level); only the regional sum is fixed.

## Nutrient Recycling to Module 50

### q18_res_recycling_nr (`flexreg_apr16/equations.gms:90-96`)

```gams
vm_res_recycling(i2,"nr") =e=
    sum(kcr, v18_res_ag_recycling(i2,kcr,"nr")
            + vm_res_ag_burn(i2,kcr,"nr")*(1-f18_res_combust_eff(kcr))
            + vm_res_biomass_bg(i2,kcr,"nr"));
```

N recycled to soil (`vm_res_recycling(i,"nr")`) has three sources:
- AG residues left on field (full N content)
- N remaining in ash from burning: fraction `(1 - f18_res_combust_eff(kcr))` of burned residue N
- All BG residue N (roots remain in soil in all cases)

### q18_res_recycling_pk (`flexreg_apr16/equations.gms:104-110`)

```gams
vm_res_recycling(i2,pk18) =e=
    sum(kcr, v18_res_ag_recycling(i2,kcr,pk18)
           + vm_res_ag_burn(i2,kcr,pk18));
```

P and K are non-volatile; 100% of burned residue P/K returns to soil (no combustion efficiency loss). BG residues are not tracked separately for P/K.

**Link to module 50**: `vm_res_recycling(i,npk)` is consumed by module 50 (Nr Soil Budget, realization `macceff_aug22/equations.gms:39`) as the recycled nutrient input to the soil nitrogen budget. This is one of the primary channels by which residue management affects soil nutrient availability and downstream N2O emissions.

## Harvest Costs

### q18_cost_prod_res (`flexreg_apr16/equations.gms:116-119`)

```gams
vm_cost_prod_kres(i2,kres) =e=
    vm_prod_reg(i2,kres) * fm_attributes("wm",kres) * f18_fac_req_kres(kres);
```

Residue harvest costs (`vm_cost_prod_kres(i,kres)`) feed into module 11 (Costs) and enter the optimization objective. Units: mio. USD17MER/yr.
