# reference/dependency_analysis/ — ⏸ Oct-2025 SNAPSHOT (not live)

**Status (2026-05-25, R25 follow-up to R6 audit)**: This directory contains a one-shot dependency-graph analysis of MAgPIE from **2025-10-10**, commit `beb125fe0`. It is NOT regenerated. Numbers, hub rankings, and cycle counts inside are correct as of that snapshot date.

Per R6 audit Cluster: presenting this as living documentation was a drift hazard — R25 follow-up downgraded the directory with this explicit snapshot label.

## When to read these files

Use the snapshot when you want:
- A static reference for the dependency-graph shape circa Oct 2025
- The `.dot` graphs (`core_dependencies.dot`, `full_dependencies.dot`, etc.) for visualization — module relationships change slowly enough that the rough topology is still useful

Use the snapshot **with caution** when you want:
- Exact interface-variable counts — verify against current `declarations.gms` (use `scripts/check_consumer_attribution.py` to recompute)
- Module centrality rankings — newer PRs (e.g. PR #866 bilateral trade overhaul) may have shifted edges
- Circular-dependency listing — see `cross_module/circular_dependency_resolution.md` for the maintained version

**Do NOT read** for:
- Current-state module-by-module dependency claims — use `core_docs/Module_Dependencies.md` (maintained) instead

## Regeneration

This snapshot was a one-shot deliverable from a v1.0-era analysis. The maintenance burden of a regenerated dependency-graph is presently handled by:
- `core_docs/Module_Dependencies.md` (maintained narrative)
- `cross_module/circular_dependency_resolution.md` (maintained cycle list)
- `scripts/check_consumer_attribution.py` (on-demand validation of consumer claims)

If a future need arises to regenerate the full graph (e.g. for a paper or release artifact), reuse the original analysis script (preserved in `dependency_analysis_raw.tar.gz`) and timestamp the new snapshot under a dated subdirectory.
